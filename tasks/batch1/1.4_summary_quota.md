# Task 1.4: Smart Summary Quota System

## Context
Free users should get 3 summaries/day. Pro users get unlimited.

## Requirements
1. Add to `app/models.py`:
   ```python
   class SummaryUsage(Base):
       __tablename__ = "summary_usage"
       id = Column(Integer, primary_key=True)
       user_id = Column(Integer, nullable=False, index=True)
       date = Column(String(10), nullable=False)  # "2026-04-21"
       count = Column(Integer, default=0)
   ```
2. Create `app/quotas.py`:
   ```python
   def can_generate_summary(user_id: int, role: str) -> bool:
       """Check if user can generate a summary today."""
       ...
   
   def record_summary_usage(user_id: int) -> None:
       """Increment today's summary usage for user."""
       ...
   
   def get_remaining_summaries(user_id: int, role: str) -> int:
       """Get remaining summaries for today."""
       ...
   ```
3. Update `/api/decisions/{id}/summary` endpoint:
   - Check quota before generating
   - Return 429 with upgrade message if quota exceeded
   - Include `remaining` in response header
4. Free tier: 3/day. Pro/Team/Enterprise: unlimited (-1)

## Acceptance
- Quota is enforced per user per day
- Quota resets at midnight UTC
- Correct HTTP 429 with upgrade CTA when exceeded
- Has a test in `tests/test_quotas.py`
