# Task 1.5: API Rate Limiting by Tier

## Context
Free: 10 API calls/day. Pro: 100/day. Team: unlimited.

## Requirements
1. Add to `app/models.py`:
   ```python
   class ApiUsage(Base):
       __tablename__ = "api_usage"
       id = Column(Integer, primary_key=True)
       user_id = Column(Integer, nullable=False, index=True)
       date = Column(String(10), nullable=False)
       endpoint = Column(String(128))
       count = Column(Integer, default=0)
   ```
2. Create `app/rate_limiter.py`:
   ```python
   from fastapi import Request, HTTPException
   
   TIER_LIMITS = {
       "free": 10,
       "pro": 100,
       "team": -1,  # unlimited
       "enterprise": -1,
   }
   
   async def rate_limit_middleware(request: Request, call_next):
       """FastAPI middleware that checks rate limits."""
       ...
   ```
3. Apply middleware in `main.py` to all `/api/*` routes (except `/api/auth/*`, `/api/health`)
4. Return 429 with `X-RateLimit-Remaining` and `X-RateLimit-Reset` headers

## Acceptance
- Limits enforced per user per day
- Auth and health routes exempt
- Headers present in all API responses
- Has a test in `tests/test_rate_limiter.py`
