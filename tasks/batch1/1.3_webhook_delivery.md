# Task 1.3: Webhook Delivery Worker

## Context
`app/slack.py` has `send_webhook()` but it's only called manually via API. Need automatic webhook delivery when alerts fire.

## Requirements
1. Add a new column `webhook_url` to `AlertRule` model (JSON metadata is fine)
2. Create `app/webhook_worker.py` with:
   ```python
   async def deliver_webhooks_for_alert(alert_id: int):
       """Deliver alert to all configured webhooks for the rule."""
       ...
   
   def run_webhook_delivery(limit: int = 50):
       """Find undelivered alerts and send webhooks."""
       ...
   ```
3. Add a `webhook_delivered` boolean column to `Alert` model
4. Call `deliver_webhooks_for_alert()` inside the alert pipeline after generating alerts
5. Add retry logic: 3 attempts with exponential backoff (5s, 15s, 45s)

## Acceptance
- Webhooks fire automatically when alerts are created
- Failed webhooks are retried 3 times
- Delivery status tracked in DB
- Has a test in `tests/test_webhook_worker.py`
