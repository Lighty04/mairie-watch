# Task 2.4: Webhook Configuration UI

## Context
Users need to configure webhook URLs for their alert rules.

## Requirements
1. Update `app/models.py` AlertRule:
   ```python
   webhook_url = Column(String(512))
   slack_webhook_url = Column(String(512))
   ```
2. Update `/api/alert-rules/advanced` endpoint to accept `webhook_url` and `slack_webhook_url` fields
3. Update `app/templates/alerts.html` to show:
   - Webhook URL input (with test button that calls `/api/webhook/test`)
   - Slack webhook URL input (with test button that calls `/api/slack/test`)
   - Delivery history/status for each rule
4. Show delivery status (last delivered, failed count) per rule

## Acceptance
- Webhook URLs stored per alert rule
- Test buttons work and show success/failure
- Delivery status visible in UI
- Mobile responsive
